Optimise
Optimise the store by removing old actions. The cloud store keeps a history of all changes from all clients, so that every client has a chance to see every change, however long it ihas been since it last updated. This history can get quite long, so you may want to optimise it by removing this history. You should only do this if you are 100% certain that no client has local changes that may need to be saved to the cloud.

This dialog lets you access the raw JSON of the database. It can be used for:

  • Making a plain-text copy of the database e.g. for backup
  • Making specific edits
  • Bulk adding new top-level folders and values

Plain text versions of the database must be secured carefully to prevent loss of sensitive data.

Note that loading the JSON will only load folders that are not already in the database.
